¿Cuáles principios de DOO has aplicado al resolver el ejercicio?
JUSTIFICAR NOMBRANDO cada principio:

DRY: Al reutilizar el código en el método ejecutarMySQL(), de modo que evito copiar y pegar código que ya esta creado.
SoC: Creando los metodos prestarLibro() y crearTablaLibro() dividiendo las responsabilidades.
YAGNI: Haciendo el minimo código necesario.
KISS: El código es simple.

Encapsulamiento: Los atributos estan protegidos con el modificador de acceso private.

Experto en información: Cada clase tiene los atributos necesarios para hacer su trabajo.
Bajo Acoplamiento: No hay clases que tengan mas de 3 clases relacionadas.
Alta cohesión: La clase LibroPrestado tiene un nombre significativo al igual que los metodos utilizados(prestarLibro(),crearTablaLibro)
Creador: Hay clases que instancian objetos de otras clases. No es responsabilidad de una clase instanciarse o crear
objetos de esa misma clase.
Controlador: Los metodos creados(prestarLibro(), crearTablaLibro()) son manejados desde la clase que funciona como controlador.
